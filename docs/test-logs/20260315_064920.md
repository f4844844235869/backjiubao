## 关键接口响应记录

以下记录聚焦本次新增的 token 安全场景，以及当前统一响应规范下的鉴权返回。

| 场景 | 方法 | 接口 | HTTP 状态 | 业务码 | 返回消息 |
| --- | --- | --- | --- | --- | --- |
| 无 token 访问当前用户 | `GET` | `/api/v1/auth/me` | `401` | `HTTP_ERROR` | `Not authenticated` |
| 非法 token 访问当前用户 | `GET` | `/api/v1/auth/me` | `401` | `AUTH_INVALID_TOKEN` | `登录状态无效或已过期` |
| 过期 token 访问当前用户 | `GET` | `/api/v1/auth/me` | `401` | `AUTH_INVALID_TOKEN` | `登录状态无效或已过期` |

| 小程序模拟登录 | `POST` | `/api/v1/auth/miniapp/code-login` | `200` | `SUCCESS` | `小程序登录成功` |

## 当前结论

- 无 token、非法 token、过期 token 三类典型鉴权失败场景已经补齐自动化测试
- 无 token 目前由 `OAuth2PasswordBearer` 先拦截，返回 `401 + HTTP_ERROR`
- 非法 token 和过期 token 会进入统一鉴权解析逻辑，返回 `401 + AUTH_INVALID_TOKEN`
- 现有统一响应结构在成功和失败场景下都保持稳定
