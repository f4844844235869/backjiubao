# 1. 文档目标

本文档用于补充商品中心的数据库表设计，细化到：

- 字段名称
- 字段含义
- 字段类型建议
- 是否必填
- 默认值建议
- 唯一性 / 约束建议
- 索引建议

当前版本仅覆盖基础商品能力，不包含套餐能力。

# 2. 设计约定

## 2.1 主键
所有表默认使用：
- `id`：`bigint` 或 `uuid`

如果你偏向业务系统可读性和简化 SQL，推荐：
- `bigint` 自增主键

## 2.2 时间字段
所有主数据表建议统一有：
- `created_at`
- `updated_at`

可选：
- `created_by`
- `updated_by`

## 2.3 逻辑删除
涉及主数据的表建议统一使用：
- `is_deleted`：`boolean`

避免物理删除导致历史单据失真。

## 2.4 状态字段
如果有“启用/停用”概念，建议：
- `is_active`：是否启用
- `is_deleted`：是否逻辑删除

## 2.5 金额字段
价格、成本、金额相关字段建议统一使用：
- `numeric(18,2)`

如果未来涉及更复杂计量，也可考虑：
- `numeric(18,4)`

---

# 3. 枚举值建议

## 3.1 商品类型 `product_type`
建议枚举：
- `NORMAL`：普通商品
- `SERVICE`：服务商品
- `FEE`：费用商品
- `VIRTUAL`：虚拟商品

## 3.2 资金限制 `fund_usage_type`
建议枚举：
- `CASH_OR_PRINCIPAL_ONLY`
- `GIFT_ONLY`
- `ALL_ALLOWED`
- `NO_MEMBER_BALANCE`
- `OFFLINE_ONLY`

## 3.3 业绩类型 `commission_type`
建议枚举：
- `NONE`
- `FIXED`
- `RATIO`

## 3.4 库存模式 `inventory_mode`
建议枚举：
- `NONE`
- `DIRECT`
- `MAPPING`

# 4. 表结构说明

# 4.1 `product_categories`

## 4.1.1 表用途
商品分类表，用于组织商品层级、菜单展示和配置管理。

## 4.1.2 字段说明

| 字段名 | 类型建议 | 必填 | 默认值 | 说明 |
|---|---|---:|---|---|
| id | bigint | 是 | 自增 | 主键 |
| parent_id | bigint | 否 | null | 父分类ID，一级分类为空 |
| code | varchar(64) | 是 |  | 分类编码 |
| name | varchar(128) | 是 |  | 分类名称 |
| level | smallint | 是 | 1 | 分类层级，建议 1/2 |
| sort_order | integer | 是 | 0 | 排序值，越小越靠前 |
| is_active | boolean | 是 | true | 是否启用 |
| is_deleted | boolean | 是 | false | 是否逻辑删除 |
| created_at | timestamp | 是 | now() | 创建时间 |
| updated_at | timestamp | 是 | now() | 更新时间 |

## 4.1.3 约束建议
- `code` 唯一
- `name + parent_id` 可考虑唯一，避免同层同名分类
- `level` 限制只允许 1 或 2

## 4.1.4 索引建议
- `idx_product_categories_parent_id`
- `idx_product_categories_is_active`
- `idx_product_categories_is_deleted`

---

# 4.2 `products`

## 4.2.1 表用途
商品主档表，保存商品基础信息和默认业务规则。

## 4.2.2 字段说明

| 字段名 | 类型建议 | 必填 | 默认值 | 说明 |
|---|---|---:|---|---|
| id | bigint | 是 | 自增 | 主键 |
| code | varchar(64) | 是 |  | 商品编码 |
| name | varchar(128) | 是 |  | 商品名称 |
| display_name | varchar(128) | 否 | null | 展示名称，用于前台/小程序 |
| print_name | varchar(64) | 否 | null | 打印名称，用于小票/后厨打印 |
| category_id | bigint | 是 |  | 分类ID |
| product_type | varchar(32) | 是 | NORMAL | 商品类型 |
| brand_id | bigint | 否 | null | 品牌ID，当前可先预留 |
| series_name | varchar(128) | 否 | null | 系列名称 |
| description | text | 否 | null | 商品描述 |
| unit | varchar(32) | 是 | 个 | 默认销售单位 |
| suggested_price | numeric(18,2) | 否 | null | 建议零售价/参考价 |
| default_fund_usage_type | varchar(32) | 是 | ALL_ALLOWED | 默认资金限制 |
| is_inventory_item | boolean | 是 | false | 是否参与库存 |
| is_commission_enabled | boolean | 是 | false | 是否参与业绩 |
| default_commission_type | varchar(16) | 是 | NONE | 默认业绩类型 |
| default_commission_value | numeric(18,2) | 是 | 0 | 默认业绩值 |
| is_profit_enabled | boolean | 是 | false | 是否参与利润分析 |
| standard_cost_price | numeric(18,2) | 否 | null | 标准成本价，仅参考 |
| is_storable | boolean | 是 | false | 是否允许存酒 |
| is_gift_allowed | boolean | 是 | false | 是否允许赠送 |
| is_min_consumption_eligible | boolean | 是 | true | 是否计入低消 |
| is_front_visible | boolean | 是 | true | 是否前台可见 |
| is_pos_visible | boolean | 是 | true | 是否 POS 可见 |
| is_sale_enabled | boolean | 是 | true | 是否允许销售 |
| is_active | boolean | 是 | true | 是否启用 |
| is_deleted | boolean | 是 | false | 是否逻辑删除 |
| remark | varchar(512) | 否 | null | 备注 |
| created_at | timestamp | 是 | now() | 创建时间 |
| updated_at | timestamp | 是 | now() | 更新时间 |

## 4.2.3 约束建议
- `code` 唯一
- `name` 可考虑唯一
- `default_commission_type = NONE` 时，`default_commission_value = 0`
- `suggested_price >= 0`
- `standard_cost_price >= 0`

## 4.2.4 索引建议
- `idx_products_category_id`
- `idx_products_product_type`
- `idx_products_is_active`
- `idx_products_is_deleted`
- `idx_products_is_sale_enabled`

# 4.3 `product_skus`

## 4.3.1 表用途
SKU 表，保存实际销售规格及 SKU 级规则。

## 4.3.2 字段说明

| 字段名 | 类型建议 | 必填 | 默认值 | 说明 |
|---|---|---:|---|---|

| product_id | bigint | 是 |  | 商品ID |
| code | varchar(64) | 是 |  | SKU 编码 |
| name | varchar(128) | 是 |  | SKU 名称 |
| spec_text | varchar(256) | 否 | null | 规格描述，如 330ml/大份 |
| suggested_price | numeric(18,2) | 否 | null | SKU 建议价 |
| barcode | varchar(64) | 否 | null | 条码 |
| is_default | boolean | 是 | false | 是否默认 SKU |
| fund_usage_type | varchar(32) | 否 | null | SKU 级资金限制，空则继承商品 |
| is_inventory_item | boolean | 否 | null | SKU 是否参与库存，空则继承商品 |
| is_commission_enabled | boolean | 否 | null | SKU 是否参与业绩，空则继承商品 |
| commission_type | varchar(16) | 否 | null | SKU 级业绩类型，空则继承商品 |
| commission_value | numeric(18,2) | 否 | null | SKU 级业绩值 |
| is_profit_enabled | boolean | 否 | null | SKU 是否参与利润分析 |
| standard_cost_price | numeric(18,2) | 否 | null | SKU 标准成本价 |
| inventory_mode | varchar(16) | 是 | NONE | 库存模式 |
| allow_negative_inventory | boolean | 是 | false | 是否允许负库存 |
| is_sale_enabled | boolean | 是 | true | 是否允许销售 |
| is_active | boolean | 是 | true | 是否启用 |
| is_deleted | boolean | 是 | false | 是否逻辑删除 |
| created_at | timestamp | 是 | now() | 创建时间 |
| updated_at | timestamp | 是 | now() | 更新时间 |

## 4.3.3 约束建议
- `code` 唯一
- `product_id + name` 建议唯一
- `suggested_price >= 0`
- `standard_cost_price >= 0`
- `inventory_mode` 必须在预设枚举中

## 4.3.4 索引建议
- `idx_product_skus_product_id`
- `idx_product_skus_is_default`
- `idx_product_skus_is_active`
- `idx_product_skus_is_deleted`
- `idx_product_skus_is_sale_enabled`

# 4.4 `store_products`

## 4.4.1 表用途
门店商品上架配置表，表示某商品在某门店是否启用、是否展示。

## 4.4.2 字段说明

| 字段名 | 类型建议 | 必填 | 默认值 | 说明 |
|---|---|---:|---|---|
| id | bigint | 是 | 自增 | 主键 |
| store_id | bigint | 是 |  | 门店ID |
| product_id | bigint | 是 |  | 商品ID |
| is_enabled | boolean | 是 | true | 门店内是否启用该商品 |
| is_visible | boolean | 是 | true | 门店前台是否可见 |
| sort_order | integer | 是 | 0 | 门店内排序 |
| created_at | timestamp | 是 | now() | 创建时间 |
| updated_at | timestamp | 是 | now() | 更新时间 |

## 4.4.3 约束建议
- `store_id + product_id` 唯一

## 4.4.4 索引建议
- `idx_store_products_store_id`
- `idx_store_products_product_id`
- `idx_store_products_is_enabled`

---

# 4.5 `store_product_skus`

## 4.5.1 表用途
门店 SKU 销售配置表，决定门店实际卖价和门店级销售规则。

## 4.5.2 字段说明

| 字段名 | 类型建议 | 必填 | 默认值 | 说明 |
|---|---|---:|---|---|
| id | bigint | 是 | 自增 | 主键 |
| store_id | bigint | 是 |  | 门店ID |
| product_id | bigint | 是 |  | 商品ID |
| sku_id | bigint | 是 |  | SKU ID |
| sale_price | numeric(18,2) | 是 | 0 | 门店实际售价 |
| list_price | numeric(18,2) | 否 | null | 门店划线价/参考价 |
| fund_usage_type | varchar(32) | 否 | null | 门店 SKU 级资金限制，空则继承 |
| is_sale_enabled | boolean | 是 | true | 门店内 SKU 是否可销售 |
| is_visible | boolean | 是 | true | 门店内 SKU 是否可见 |
| is_default | boolean | 是 | false | 是否门店默认销售 SKU |
| effective_from | timestamp | 否 | null | 生效开始时间 |
| effective_to | timestamp | 否 | null | 生效结束时间 |
| created_at | timestamp | 是 | now() | 创建时间 |
| updated_at | timestamp | 是 | now() | 更新时间 |

## 4.5.3 约束建议
- `store_id + sku_id` 唯一
- `sale_price >= 0`
- `list_price >= 0`
- `effective_to > effective_from`（若同时存在）

## 4.5.4 索引建议
- `idx_store_product_skus_store_id`
- `idx_store_product_skus_product_id`
- `idx_store_product_skus_sku_id`
- `idx_store_product_skus_is_sale_enabled`
- `idx_store_product_skus_effective_from_effective_to`

---

# 4.6 `sku_inventory_mappings`

## 4.6.1 表用途
SKU 到库存扣减规则映射表，用于定义销售 1 个 SKU 时，需要扣哪些库存、扣多少。

## 4.6.2 字段说明

| 字段名 | 类型建议 | 必填 | 默认值 | 说明 |
|---|---|---:|---|---|
| id | bigint | 是 | 自增 | 主键 |
| sku_id | bigint | 是 |  | 销售 SKU ID |
| inventory_product_id | bigint | 否 | null | 对应库存商品ID |
| inventory_sku_id | bigint | 否 | null | 对应库存 SKU ID |
| deduct_quantity | numeric(18,4) | 是 | 1 | 扣减数量 |
| deduct_unit | varchar(32) | 是 | 个 | 扣减单位 |
| sort_order | integer | 是 | 0 | 顺序 |
| created_at | timestamp | 是 | now() | 创建时间 |
| updated_at | timestamp | 是 | now() | 更新时间 |

## 4.6.3 约束建议
- `deduct_quantity > 0`
- `sku_id` 必须存在
- `inventory_product_id` / `inventory_sku_id` 至少有一个有效
- 若同一个 SKU 对应多个扣减项，则允许多行

## 4.6.4 索引建议
- `idx_sku_inventory_mappings_sku_id`
- `idx_sku_inventory_mappings_inventory_sku_id`

# 5. 订单快照字段建议（供下游表设计参考）

| 字段名 | 类型建议 | 说明 |
|---|---|---|
| product_id | bigint | 商品ID |
| sku_id | bigint | SKU ID |
| category_id_snapshot | bigint | 下单时分类ID |
| category_name_snapshot | varchar(128) | 下单时分类名称 |
| product_name_snapshot | varchar(128) | 下单时商品名称 |
| sku_name_snapshot | varchar(128) | 下单时 SKU 名称 |
| product_code_snapshot | varchar(64) | 下单时商品编码 |
| sku_code_snapshot | varchar(64) | 下单时 SKU 编码 |
| unit_price_snapshot | numeric(18,2) | 下单时成交单价 |
| store_sale_price_snapshot | numeric(18,2) | 门店售价快照 |
| fund_usage_type_snapshot | varchar(32) | 下单时资金限制 |
| is_inventory_item_snapshot | boolean | 下单时是否走库存 |
| commission_type_snapshot | varchar(16) | 下单时业绩类型 |
| commission_value_snapshot | numeric(18,2) | 下单时业绩值 |

# 7. 关键唯一约束建议汇总

## 分类
- `product_categories.code` 唯一

## 商品
- `products.code` 唯一
- `products.name` 可考虑唯一

## SKU
- `product_skus.code` 唯一
- `product_skus(product_id, name)` 建议唯一

## 门店商品配置
- `store_products(store_id, product_id)` 唯一

## 门店 SKU 配置
- `store_product_skus(store_id, sku_id)` 唯一

---

# 8. 关键索引建议汇总

建议至少建以下索引：

- `product_categories(parent_id)`
- `products(category_id)`
- `products(is_active, is_deleted)`
- `product_skus(product_id)`
- `product_skus(is_active, is_deleted)`
- `store_products(store_id, is_enabled)`
- `store_product_skus(store_id, is_sale_enabled)`
- `sku_inventory_mappings(sku_id)`

如果后续商品数量和门店数量增大，再补组合索引。

# 10. 一句话总结

这套表结构的核心目标是：

- 商品负责归类
- SKU 负责成交
- 门店 SKU 负责最终售价
- SKU 库存映射负责扣减规则
- 历史单据靠快照兜底
- 主数据允许改，但历史数据不漂
